# Настройка Vercel для деплоя

## Проблема: Быстрая сборка (98ms)

Если сборка завершается слишком быстро, это означает, что Vercel не находит папку `Frontend` или не выполняет правильную сборку.

## Решение 1: Настройка через веб-интерфейс Vercel (Рекомендуется)

1. Перейдите в настройки проекта на Vercel
2. Откройте раздел **Settings → General**
3. Найдите **Root Directory**
4. Нажмите **Edit**
5. Выберите папку `Frontend`
6. Сохраните изменения
7. Перезапустите деплой (Redeploy)

## Решение 2: Использование vercel.json в корне

В корне репозитория создан файл `vercel.json`, который указывает Vercel использовать папку `Frontend` как root directory.

Если это не работает, удалите `vercel.json` из корня и используйте настройки через веб-интерфейс (Решение 1).

## Решение 3: Перемещение vercel.json

Если у вас монорепозиторий, убедитесь, что `vercel.json` находится в папке `Frontend`, а в настройках Vercel указан Root Directory как `Frontend`.

## Проверка правильной сборки

После правильной настройки вы должны увидеть в логах:

```
Running "npm install"
...
Running "npm run build"
...
Build Completed in /vercel/output [XXs] (не 98ms!)
```

## Переменные окружения

Не забудьте добавить в настройках Vercel:

- **Name**: `NEXT_PUBLIC_API_URL`
- **Value**: URL вашего бэкенда (например, `https://your-app.railway.app`)
- **Environment**: Production, Preview, Development

## После настройки

1. Убедитесь, что Root Directory установлен как `Frontend`
2. Добавьте переменную окружения `NEXT_PUBLIC_API_URL`
3. Перезапустите деплой
4. Проверьте логи - сборка должна занимать больше времени (обычно 1-3 минуты)
