# Устранение неполадок бота

## Бот не отвечает на команды

### 1. Проверьте, что бот запущен

Убедитесь, что бот запущен в терминале:

```bash
cd TelegramBot
python main.py
```

Вы должны увидеть сообщение:
```
INFO:__main__:Загружен файл переменных окружения: C:\Users\Andrey\Desktop\XAKATON\phoenix_lab\Backend\BOT_TOKEN.env
INFO:__main__:Бот запущен
INFO:__main__:Настроено каналов: X
```

### 2. Проверьте токен бота

Убедитесь, что файл `Backend/BOT_TOKEN.env` содержит правильный токен:

```
BOT_TOKEN=8244129596:AAGX-V-5hzHPjTyuZVioZaReTWyPiXwSptM
```

### 3. Проверьте логи на наличие ошибок

При запуске бота проверьте, нет ли ошибок в консоли. Типичные ошибки:

- `BOT_TOKEN не найден` - проверьте путь к файлу с токеном
- `Connection error` - проблемы с интернет-соединением
- `Unauthorized` - неверный токен бота

### 4. Проверьте, что бот активен в BotFather

Откройте BotFather в Telegram и отправьте команду `/mybots`. Убедитесь, что ваш бот активен.

### 5. Перезапустите бота

Если бот не отвечает:

1. Остановите бота (Ctrl+C)
2. Подождите 5 секунд
3. Запустите снова: `python main.py`

### 6. Проверьте обработчик команды /start

Бот должен отвечать на команду `/start` даже без аргументов. Если не отвечает, проверьте логи на наличие ошибок.

## Типичные ошибки

### ModuleNotFoundError: No module named 'aiogram'

Установите зависимости:
```bash
pip install -r requirements.txt
```

### BOT_TOKEN не найден

Убедитесь, что файл `Backend/BOT_TOKEN.env` существует и содержит токен.

### Бот отвечает, но не обрабатывает команды

Проверьте, что обработчики зарегистрированы правильно. Убедитесь, что используется `CommandStart()` для команды `/start`.
