# Исправление ошибки "Username invalid"

## Причины ошибки:

1. **Домен не настроен в BotFather** (самая частая причина)
2. Неправильное имя бота (регистр или формат)
3. Виджет не может загрузиться из-за проблем с доменом

## Решение:

### Шаг 1: Настройка домена в BotFather

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду: `/setdomain`
3. Выберите вашего бота `Phoenix_LogIN_bot` из списка
4. Укажите домен:
   - Для локальной разработки: `localhost` или `127.0.0.1`
   - Для продакшена: ваш реальный домен (например: `yourdomain.com`)

**Важно:** После настройки домена подождите 1-2 минуты, чтобы изменения вступили в силу.

### Шаг 2: Проверка имени бота

Убедитесь, что имя бота в коде точно совпадает с именем, которое вы видите в Telegram:
- Откройте вашего бота в Telegram
- Имя должно быть БЕЗ символа `@`
- Регистр важен! Если бот называется `Phoenix_LogIN_bot`, то именно так и должно быть в коде

### Шаг 3: Проверка в консоли браузера

1. Откройте консоль разработчика (F12)
2. Перейдите на вкладку Console
3. Откройте меню авторизации
4. Проверьте, нет ли ошибок загрузки скрипта

### Шаг 4: Альтернативный способ проверки

Если виджет все еще не работает, попробуйте:

1. Проверить, что сайт открыт по правильному адресу:
   - Если домен настроен как `localhost`, сайт должен быть на `http://localhost:3000`
   - Если домен настроен как `127.0.0.1`, сайт должен быть на `http://127.0.0.1:3000`

2. Очистить кеш браузера (Ctrl+Shift+Delete)

3. Попробовать в режиме инкогнито

## Проверка правильности настройки:

После настройки домена в BotFather, вы должны увидеть сообщение:
```
Success! Domain updated. The widget will be available on: yourdomain.com
```

## Если ничего не помогает:

1. Убедитесь, что бот активен (отправьте ему `/start`)
2. Проверьте, что имя бота в коде точно совпадает с реальным именем
3. Попробуйте создать нового бота и использовать его имя
4. Для локальной разработки можно использовать ngrok для создания публичного URL
