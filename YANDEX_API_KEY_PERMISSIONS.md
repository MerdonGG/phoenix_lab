# Выбор разрешений для API ключа Yandex Cloud

## Проблема:
В списке нет облаков - это нормально, это список **разрешений** (permissions), а не облаков.

## Что выбрать:

### Вариант 1: Для работы с AI Assistant через MCP Gateway

Выберите разрешение:
```
yc.serverless.mcpGateways.invoke
```

Это разрешение для вызова MCP Gateway (ваш сервер "phoenix").

### Вариант 2: Для работы с Language Models напрямую

Выберите разрешение:
```
yc.ai.languageModels.execute
```

Это разрешение для выполнения запросов к языковым моделям.

### Вариант 3: Оба разрешения (рекомендуется)

Выберите оба:
- `yc.serverless.mcpGateways.invoke`
- `yc.ai.languageModels.execute`

## Как выбрать:

1. **В поле поиска** введите: `mcp` или `languageModels`
2. **Найдите нужное разрешение** в списке
3. **Кликните на него** - оно добавится в выбранные

## Где выбрать облако/проект:

Поле "Область действия" (Scope) - это **отдельное поле выше**, не в этом списке разрешений.

Если поле "Область действия" пустое или не показывает ваши облака:

1. **Проверьте, что вы находитесь в правильном облаке:**
   - В верхней части страницы должно быть указано ваше облако (например: `cloud-ivancorpusov`)
   - Если нет - переключитесь на нужное облако

2. **Попробуйте создать ключ на уровне проекта:**
   - Перейдите в конкретный проект
   - Создайте ключ там

3. **Или оставьте поле пустым:**
   - Некоторые ключи можно создать без указания области действия
   - Они будут работать для всех ресурсов, к которым у вас есть доступ

## После выбора разрешений:

1. Выберите нужные разрешения из списка
2. Заполните поле "Область действия" (если возможно) или оставьте пустым
3. Нажмите "Создать"
4. Скопируйте ключ

## Альтернатива: Создать ключ без указания области действия

Если не можете выбрать область действия:
- Оставьте поле "Область действия" пустым
- Выберите разрешения: `yc.serverless.mcpGateways.invoke` или `yc.ai.languageModels.execute`
- Создайте ключ
- Ключ будет работать для всех ресурсов, к которым у вас есть доступ
