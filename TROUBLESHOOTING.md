# Диагностика проблем

## Быстрая проверка:

### 1. Бэкенд запущен?
Проверьте, что в терминале видно:
```
* Running on http://127.0.0.1:5000
```

### 2. API ключ загружен?
В логах бэкенда должно быть:
```
INFO:__main__:Yandex Cloud API клиент инициализирован
```

Если видите:
```
WARNING:__main__:YANDEX_CLOUD_API_KEY не найден
```
→ Перезапустите бэкенд после добавления ключа в BOT_TOKEN.env

### 3. CORS ошибка?
Если в консоли браузера видите CORS ошибку:
→ Перезапустите бэкенд (Ctrl+C, затем `python server.py`)

### 4. Какая конкретная ошибка?

**Откройте консоль браузера (F12) и посмотрите:**
- Какая ошибка в консоли?
- Какой статус ответа (200, 400, 500)?
- Что в Network tab при запросе?

**Проверьте логи бэкенда:**
- Какие ошибки в терминале бэкенда?
- Есть ли запросы к `/api/rewrite-article`?

## Частые проблемы:

### Проблема 1: "Yandex Cloud API не настроен"
**Решение:** 
1. Проверьте файл `Backend/BOT_TOKEN.env` - есть ли там `YANDEX_CLOUD_API_KEY`?
2. Перезапустите бэкенд

### Проблема 2: CORS ошибка
**Решение:**
1. Перезапустите бэкенд
2. Обновите страницу (Ctrl+F5)

### Проблема 3: "Ошибка обработки статьи через Yandex Cloud API"
**Решение:**
1. Проверьте логи бэкенда - там будет детальная ошибка
2. Возможно, неправильный ID инструмента или формат запроса
3. Проверьте, что разрешения для API ключа правильные

### Проблема 4: "NetworkError"
**Решение:**
1. Убедитесь, что бэкенд запущен
2. Проверьте, что порт 5000 не занят другим процессом
3. Попробуйте открыть `http://localhost:5000/api/health` в браузере

## Что проверить прямо сейчас:

1. **Бэкенд запущен?** → Откройте терминал, где запущен бэкенд
2. **Есть ли ошибки в логах?** → Посмотрите последние строки в терминале бэкенда
3. **Что в консоли браузера?** → Нажмите F12, откройте вкладку Console
4. **Что в Network?** → F12 → Network → попробуйте отправить запрос → посмотрите ответ
